## Rule

No performance, accuracy, backtest, game-count, model-result, ROI, edge, or upset-detection claim may be used in product UI, docs, investor materials, or engineering notes unless backed by:

- committed evaluation script
- accessible dataset or database artifact
- reproducible command
- generated report
- timestamp
- engine version
- feature version
- provider snapshot version

## Backtest Claim Thresholds

No public performance claim is allowed unless all of the following are true:

- `gamesEvaluated >= 1000`
- at least 3 leagues are represented
- `inputSha256` is recorded
- `engineVersion` is recorded
- `featureVersion` is recorded
- `translatorVersion` is recorded
- a report artifact exists in `data/backtest-artifacts` or approved external storage
- the report was generated by `npm run backtest:historical`

## 9,500-Game Claim Rule

A 9,500-game claim is forbidden unless:

- `gamesTotal >= 9500`
- `gamesEvaluated` is shown honestly
- `gamesSkipped` is shown honestly
- `skipReasons` are shown honestly
- feature coverage is shown honestly
- the input corpus path and SHA256 are recorded

If `gamesEvaluated < 100`, reports and UI must display:

> Sample size too small for performance claims.

## Forbidden Public Copy Without Artifact

- 9,500 recorded events
- verified 9,500-game backtest
- model accuracy
- ROI
- edge

- upset detection accuracy
- agency calibrated
